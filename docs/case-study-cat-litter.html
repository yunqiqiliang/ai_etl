AI ETL Reference Case Study

猫砂竞品主数据 AI 自动生成

从 5 个电商平台的 Excel 销售数据出发，通过云器 Lakehouse + LLM Batch 推理，自动提取 862 条竞品商品的结构化主数据（品牌、类目、规格、配比），生成与人工标注一致的标准主数据表。

📋 业务背景

某宠物用品品牌（福丸）需要建立猫砂品类的竞品价盘数据库，覆盖抖音、天猫、天猫超市、京东自营、京东POP 五个电商平台。

5

电商平台

抖音/天猫/天猫超市/京东自营/京东POP

862

竞品商品

来自各平台的猫砂类商品

46

标杆主数据

福丸品牌人工标注的SKU主数据

16

类目编码

四级类目体系(混合/植物/矿砂等)

核心挑战

各平台的商品名称是营销标题（如"福丸12包栀子花猫砂木薯猫砂混合豆腐猫砂去味除臭2.5kg"），需要从中提取结构化的产品主数据：品牌、类目、标准化产品名、配比、规格。人工处理 862 条数据耗时且不一致。

④ 合并去重 → 生成统一主数据表

5 个渠道的 AI 结果通过 SQL 合并、去重、JSON 解析，生成与 product_master 结构一致的竞品主数据表。

合并 SQL 逻辑

CREATE TABLE cat_litter.competitor_product_master AS
WITH all_data AS (
    -- 5 个渠道 UNION ALL，JOIN 回源表取原始商品名称
    SELECT '京东自营' AS platform, a.商品编号 AS source_key,
           b.商品名称 AS source_name, a.ai_result
    FROM cat_litter.jd_self_product_master a
    LEFT JOIN cat_litter.sales_jd_self b ON a.商品编号 = b.商品编号
    UNION ALL ...
),
deduped AS (
    -- 同平台同 source_key 去重
    SELECT *, ROW_NUMBER() OVER (PARTITION BY platform, source_key) AS rn
    FROM all_data
),
with_seq AS (
    -- 按品牌+类目分组编号，用于生成主数据编码
    SELECT *, ROW_NUMBER() OVER (PARTITION BY brand, cat_code ORDER BY platform) AS seq
    FROM parsed WHERE rn = 1
)
SELECT
    -- 与 product_master 对齐的 9 列
    brand AS 品牌, '宠物用品' AS 一级类目, '猫砂' AS 二级类目,
    cat3 AS 三级类目, cat4 AS 四级类目, pname AS 产品名称,
    -- 自动生成主数据编码：品牌前2字母-PS-CL-类目编码-序号-规格克数
    CONCAT(UPPER(SUBSTR(brand,1,2)),'-PS-CL-',cat_code,'-',
           LPAD(seq,4,'0'),'-',CAST(spec*1000 AS INT)) AS 主数据,
    spec AS 规格, formula AS 配比,
    -- 额外溯源字段
    platform AS 数据来源平台, source_key AS 源数据主键,
    source_name AS 源数据商品名称, cat_code AS 类目编码, ai_result AS ai_raw
FROM with_seq;

💡 关键经验

1. Few-shot 示例决定质量

福丸的 46 条人工标注数据不是"要处理的数据"，而是"教 AI 怎么做的示例"。3 个精选的 few-shot 示例（覆盖不同品牌和类目）比 10 个同质示例效果更好。

2. 类目体系必须封闭

把 16 个类目编码作为"选择题"嵌入 prompt，而不是让 AI 自由发挥。这确保了 100% 的类目匹配率，且与 category_mapping 表完全对齐。

3. 低 temperature 保证结构化输出

temperature=0.1 确保 AI 输出稳定的 JSON 格式。temperature 过高会导致 JSON 格式不一致或添加多余的解释文字。

4. 并行提交节省时间

5 个渠道的 batch 同时提交，在 DashScope 服务端并行处理。总等待时间 = 最慢的那个渠道，而不是 5 个渠道之和。

5. 溯源字段不可少

保留 platform、source_key、source_name、ai_raw 等溯源字段，方便后续人工抽检、问题排查和数据回溯。

6. 规格提取是短板

27% 的商品无法从名称中推断规格（名称中不含重量信息）。后续可结合商品详情页或价格信息补充。

🔧 复现指南

按以下步骤可以在你自己的数据上复现整个流程。

准备数据

将 Excel 数据导入云器 Lakehouse（可用 Lakehouse Studio 上传，或用 Python 脚本）。确保有一张"标杆主数据"表作为 few-shot 示例。

安装 AI ETL

git clone https://github.com/yunqiqiliang/ai_etl.git
cd ai_etl && pip install -e .

用 plan 命令生成配置

python -m ai_etl plan --table your_schema.your_table --hint "提取品牌、类目、规格"

AI 会采样数据并推荐 model、prompt、参数。将输出的 config 片段保存为 yaml 文件。

提交 Batch 推理

python -c "
from ai_etl.config import Config
from ai_etl.pipeline import AIETLPipeline
cfg = Config(config_path=Path('configs/your_config.yaml'))
AIETLPipeline(config=cfg).run()
"

合并去重生成主数据表

用 SQL 从 AI 结果表中解析 JSON、去重、生成主数据编码，创建最终的主数据表。参考 scripts/rebuild_competitor_master_v2.py。
